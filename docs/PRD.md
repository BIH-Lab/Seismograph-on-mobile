# PRD.md — Product Requirements Document

## 제품 목적
스마트폰 내장 가속도 센서를 활용해 지진동 데이터를 실시간 수집하고,
웹 플랫폼에서 시각화·통계 처리·분석까지 가능한 연구/교육용 계측 시스템 구축.
앱 설치 없이 브라우저만으로 동작하는 것이 핵심 차별점.

## 사용자
- 연구자 (데이터 수집 및 분석 목적)
- 교사/학생 (지구과학 교육 현장 활용)

## 핵심 기능 요구사항

### Cycle 1 — Activity 1 : 지진파 색으로 보기 ✅ 완료
| 기능 | 설명 |
|------|------|
| 센서 권한 요청 | iOS/Android 브라우저에서 DeviceMotion 권한 처리 |
| 캘리브레이션 | 측정 시작 시점 5샘플 평균을 기준값으로 설정, 이후 변화량(Δ) 출력 |
| 실시간 데이터 수집 | X·Y·Z축 가속도 + 타임스탬프 수집 |
| 배경색 시각화 | Z축 진폭 → 배경색 매핑 (파랑→초록→빨강, HSL) |
| 실시간 그래프 | Z축 파형 실시간 표시 (슬라이딩 윈도우 10초, 자동 Y축 스케일) |
| 측정 시작/정지 | 버튼으로 측정 제어 |

### Cycle 2 — Activity 2 : 신호 비교하기
| 기능 | 설명 |
|------|------|
| CSV 파일 드롭 | 여러 CSV 파일을 드래그&드롭으로 불러오기 |
| 다중 파형 시각화 | 여러 기기 신호를 오버레이 비교 |
| 기본 통계 | 최대값·최소값·RMS·샘플링 레이트 자동 계산 |

### Cycle 3 — Activity 2 심화
| 기능 | 설명 |
|------|------|
| FFT 주파수 분석 | 주파수 스펙트럼 시각화 |
| 스펙트로그램 | 시간-주파수 분포 표시 |

### Cycle 4 — Activity 3·4
| 기능 | 설명 |
|------|------|
| 주시곡선 그리기 | P파·S파 도달 시간으로 진원 탐색 |
| 진원 찾기 | GPS 위치 정보를 이용해 진원지 역산 |

## 비기능 요구사항
- 모바일 퍼스트 반응형 UI
- HTTPS 필수 (센서 API 요구사항)
- iOS Safari / Android Chrome 크로스 브라우저 지원
- 센서 샘플링 레이트 : 목표 100Hz
- 데이터 정밀도 : 소수점 6자리 이상 유지
- 서버 없이 브라우저 단독 동작 (Cycle 2까지)

## 제외 범위 (논문 범위 밖)
- 푸시 알림 / 조기 경보
- 실시간 다중 기기 동기화 (Cycle 5 이후 재검토)
